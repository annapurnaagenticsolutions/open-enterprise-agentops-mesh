# v2.8 Forward Roadmap

## Recommended next release

v2.8 should be **Public Launch Candidate and GitHub Pages Finalization**.

## Why

v2.7 makes the evidence package ready. The next release should finalize the public launch candidate:

- final landing page copy,
- top-level project narrative cleanup,
- GitHub Pages publishing checklist,
- launch issue template,
- launch announcement draft,
- demo recording final QA,
- repository tag checklist.

## What v2.8 should not do

Do not add live provider calls or live connector execution. The project should be launched as a safe open-source control-plane MVP before adding higher-risk runtime capability.
