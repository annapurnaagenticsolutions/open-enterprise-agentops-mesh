# Demo Recording Quality Gate

## Gate objective

The demo should make the project understandable in under seven minutes without overselling production readiness.

## Required checks

| Check | Required outcome |
|---|---|
| Public site opens locally | Pass |
| Interactive demo path is clear | Pass |
| Procurement scenario explains value | Pass |
| Governance boundary is visible | Pass |
| Live execution is clearly disabled | Pass |
| Validation proof is available | Pass |
| Recording script has timing guidance | Pass |
| Call-to-action is precise | Pass |

## Rejection criteria

Do not record the demo if:

- live execution boundaries are unclear,
- the demo implies production readiness,
- validation evidence is missing,
- the user journey is confusing,
- the project narrative drifts into a generic chatbot framework.
