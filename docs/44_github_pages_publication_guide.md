# GitHub Pages Publication Guide

## Recommended publication structure

Use the `site/` folder as the GitHub Pages source.

## Steps

1. Create a GitHub repository.
2. Commit the full project.
3. Go to repository settings.
4. Open **Pages**.
5. Select branch: `main`.
6. Select folder: `/site`.
7. Save.
8. Open the published Pages URL.

## Content discipline

The public site should not over-explain every technical file. It should help visitors understand:

1. What problem the project solves.
2. Why AgentOps governance matters.
3. How the control plane works.
4. What the procurement accelerator demonstrates.
5. How to run the backend locally.
6. How to contribute or extend.
