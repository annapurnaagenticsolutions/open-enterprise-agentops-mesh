# v2.1 — Documentation Pruning, OpenAPI Polish, and Contributor Readiness

## Purpose

v2.1 is an adoption-quality release. It does not add risky runtime capability. It improves how a new user, contributor, architect, or reviewer understands and validates the project.

## Why this release exists

By v2.0, the project had enough breadth to be credible, but also enough documentation volume to overwhelm a first-time reader. v2.1 introduces a curated path, API catalog, contributor flow, and repository validation scripts.

## Scope

Included:

- simplified root README
- curated documentation map
- public onboarding guide
- OpenAPI-lite catalog
- contributor readiness report
- public repo validation script
- OpenAPI-lite export script
- GitHub issue templates and CI workflow
- contributor console and API catalog static pages

Excluded:

- live connector execution
- real provider execution
- real IAM integration
- raw secret handling
- production deployment automation

## Decision

v2.1 keeps the platform stable and makes it easier to publish, explain, and contribute to. This is the correct sequence before adding any v2.3 feature work.
