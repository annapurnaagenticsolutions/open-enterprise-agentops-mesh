# v2.1 Release Notes

## Release name

Documentation Pruning, OpenAPI Polish, and Contributor Readiness

## Summary

v2.1 improves adoption quality. It simplifies the public entry point, adds curated documentation, improves API discoverability, introduces contributor-readiness checks, and adds a public repo validation script.

## Added

- public repo onboarding guide
- curated documentation map
- OpenAPI usage and contract guide
- contributor workflow and quality gates
- maintainer release playbook
- issue triage and adoption model
- OpenAPI-lite catalog
- contributor readiness report
- static API catalog page
- static contributor console
- public repo validation script
- OpenAPI-lite export script
- GitHub issue templates
- GitHub Actions CI workflow
- v2.1 release scorecard

## Changed

- root README simplified around public adoption
- docs README converted into a curated index
- control-plane summary now exposes OpenAPI-lite and contributor-readiness endpoints
- smoke checks include v2.1 public adoption endpoints

## Not changed

- no live connector execution
- no live model-provider calls
- no raw secret storage
- no production IAM integration
- no irreversible enterprise writes

## Release decision

Ready for public repository publication after a final human review of copy, license, and GitHub Pages settings.
