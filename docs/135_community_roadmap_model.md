# Community Roadmap Model

## Roadmap signal types

- Use-case demand
- Architecture critique frequency
- Adoption blocker
- Documentation gap
- Benchmark gap
- Deployment friction
- Governance gap

## Roadmap decision model

```text
Signal count
+ enterprise relevance
+ open-source fit
+ implementation feasibility
+ risk reduction value
- scope expansion risk
= roadmap priority
```

## Candidate themes after v2.5

| Theme | Reason |
|---|---|
| Scenario expansion | Helps contributors understand practical use |
| Install simplification | Improves adoption conversion |
| Security review evidence | Builds enterprise trust |
| Public demo polish | Improves industry presence |
| GitHub issue automation | Useful later, but not required for v2.5 |
