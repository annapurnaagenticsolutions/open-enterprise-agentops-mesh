# v2.5 Forward Roadmap — Public Feedback Loop and Community Intake

## Recommended next release

v2.5 should add a public feedback and adoption loop rather than new runtime capability.

## Proposed scope

- Feedback issue templates
- Architecture critique template
- Use-case submission template
- Scenario contribution workflow
- Community roadmap board model
- Public changelog page
- Adoption metrics checklist
- Maintainer triage dashboard

## Why this should come next

The project now has enough internal capability. The next strategic value is external validation from architects, AI product managers, governance leaders, and developers.

## Still defer

- Live connectors
- Live model-provider execution
- Production IAM
- External vault integration
- Hosted SaaS deployment

Those should come only after public feedback confirms the most valuable production path.
