# UX Copy and Empty-State Guide

## Copy style

Use precise, enterprise-oriented language. Avoid hype.

Prefer:

- governed agent lifecycle
- deterministic control
- audit evidence
- sandbox execution
- readiness report

Avoid:

- magic automation
- autonomous everything
- production-ready without boundaries
- self-running enterprise AI

## Boundary banner

Every public demo page should include:

> Live providers, live connectors, raw secrets, and production IAM remain disabled in this release.

## Empty-state rules

Empty states should explain what action creates value. Example:

> No benchmark runs yet. Run a deterministic benchmark suite to create release evidence before adding live execution.
