# Public Site Information Architecture

## Problem

The project has many capabilities and pages. Without a clear information architecture, public visitors may see the repository as complex rather than credible.

## v2.6 information architecture

The site should support four audience paths:

1. Executive overview
2. Architecture review
3. Governance and risk review
4. Contributor path

## Required homepage structure

```text
Hero
→ Problem statement
→ Control-plane lifecycle
→ Guided demo CTA
→ Capability map
→ Release evidence
→ Contribution path
→ Boundary note
```

## Navigation rule

Do not expose every page with equal weight. Give users a default path, then let experts inspect deeper consoles.
