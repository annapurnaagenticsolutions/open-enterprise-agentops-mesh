# v2.1 Forward Roadmap

## Recommended v2.1 theme

**Documentation pruning, OpenAPI polish, and contributor readiness.**

Do not add major new runtime capabilities immediately after v2.0. The next release should improve adoption quality.

## Candidate v2.1 scope

- reduce repetitive docs
- generate a concise architecture handbook
- add issue templates mapped to capability groups
- add contributor onboarding guide
- add architecture diagrams as SVG/PNG
- add OpenAPI export script
- add API examples folder
- add deterministic demo payload collection
- add release checklist automation

## Deferred until later

- real provider execution
- real connector execution
- external secret-manager integration
- hosted UI
- database migration implementation

## Suggested v2.3+

- v2.3: SQLite/Postgres persistence adapter prototype
- v2.3: OpenTelemetry/SIEM export model
- v2.4: real provider gateway adapter in disabled-by-default mode
- v2.5: first live-candidate connector in isolated demo mode
