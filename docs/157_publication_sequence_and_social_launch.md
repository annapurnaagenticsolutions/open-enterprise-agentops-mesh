# Publication Sequence and Social Launch

## Recommended order

1. Publish the repository privately or locally validate the release candidate.
2. Run tests and smoke checks.
3. Run public repo validation.
4. Run benchmark harness.
5. Run release evidence report.
6. Run launch-candidate report.
7. Enable GitHub Pages from `/site`.
8. Record the five-minute demo.
9. Publish the GitHub repository.
10. Publish LinkedIn launch post.
11. Invite architecture critique and use-case submissions.

## Social positioning

Primary message:

> I am launching Open Enterprise AgentOps Mesh, an open-source AgentOps control plane for governing, evaluating, operating, and auditing enterprise AI agents before production.

Supporting themes:

- Enterprise AI agents need governance before scale.
- AgentOps requires evidence, auditability, policy, evaluation, and approval workflows.
- Open-source assets can help enterprises move from GenAI demos to disciplined agentic systems.

## Recommended post sequence

1. Launch announcement.
2. Architecture deep dive.
3. Governance and policy-as-code thread.
4. Procurement control-plane demo.
5. Open-source contribution and feedback invitation.
