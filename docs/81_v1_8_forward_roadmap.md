# v1.8 Forward Roadmap — Provider Gateway Governance and Model-Routing Controls

## Proposed objective

Add governance around model/provider routing before enabling real provider calls.

## Candidate capabilities

- provider gateway policy model,
- approved model catalog,
- data sensitivity to model-routing rules,
- tenant/model allow-list,
- cost ceiling controls,
- context-window and retention policy,
- prompt/data boundary rules,
- provider fallback policy,
- mock provider gateway decision API,
- audit events for routing decisions.

## Still excluded

- raw API keys,
- live model-provider execution by default,
- production data transmission,
- non-simulated enterprise connector writes.
