Integration Blueprint

Self-Healing Pipeline ChatOps Scenario Integration

Blueprint for connecting the Self-Healing Pipeline POC to the existing Teams → Azure Bot → AWS Lambda → Claude Orchestrator → MCP tools architecture, while using scenario-backed self-healing data before live tool data is ready.

Recommended Mode: scenario_backed_chatops Teams ChatOps Claude Orchestrator MCP Tool Calling

Core Correction

Do not force self-healing diagnosis on generic sample Jira/GitLab/TeamCity data.

Use curated near-real self-healing scenarios first.
Expose those scenarios through scenario-aware MCP tools.
Then gradually replace scenario-backed responses with live TeamCity/GitLab/Jira data.

What is Real in Demo

Teams interaction, Azure Bot routing, Lambda gateway, Claude orchestration, MCP tool calling, self-healing logic, policy decisions, approval UX.

What is Scenario-Backed

TeamCity logs/builds, GitLab diffs, Jira issues, pattern memory, documentation checklist data.

What is Deferred

Production auto-merge, secret mutation, rollback, Terraform/IAM/Kubernetes/database changes, unvalidated memory promotion.
