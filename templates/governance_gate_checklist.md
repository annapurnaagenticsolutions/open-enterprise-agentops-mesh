# Agent Governance Gate Checklist

## Gate 1: Use-case approval

- [ ] Business owner identified
- [ ] Business outcome defined
- [ ] Baseline metric captured
- [ ] Target metric defined
- [ ] Target users identified
- [ ] Workflow boundaries defined

Decision:

- [ ] Pass
- [ ] Conditional pass
- [ ] Fail

## Gate 2: Risk and autonomy classification

- [ ] Autonomy level assigned
- [ ] Risk level assigned
- [ ] Sensitive-data exposure reviewed
- [ ] Financial/legal/customer/employment impact reviewed
- [ ] Human approval requirements defined

Decision:

- [ ] Pass
- [ ] Conditional pass
- [ ] Fail

## Gate 3: Data readiness

- [ ] Data sources approved
- [ ] Data owners identified
- [ ] Data freshness verified
- [ ] Access controls defined
- [ ] Retrieval quality tested
- [ ] Citation policy defined

Decision:

- [ ] Pass
- [ ] Conditional pass
- [ ] Fail

## Gate 4: Architecture review

- [ ] Agent roles defined
- [ ] Model gateway defined
- [ ] Tool gateway defined
- [ ] Policy layer defined
- [ ] Observability layer defined
- [ ] Audit log defined
- [ ] Escalation path defined

Decision:

- [ ] Pass
- [ ] Conditional pass
- [ ] Fail

## Gate 5: Evaluation review

- [ ] Golden task tests passed
- [ ] Retrieval tests passed

- [ ] Cost/latency profile acceptable
- [ ] Scorecard grade meets target

- [ ] Pass

- [ ] Fail

- [ ] Pilot users defined
